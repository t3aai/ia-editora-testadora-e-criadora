# CRITÉRIOS DE VALIDAÇÃO

## 1. COMPLETUDE (20 pontos)
- Todas as seções obrigatórias estão presentes?
- Informações críticas foram preservadas?
- Novos elementos necessários foram adicionados?

## 2. CONFORMIDADE COM REGRAS (30 pontos)
- Segue 100% as regras T3A?
- Estrutura está conforme padrão?
- Formatação está impecável?

## 3. QUALIDADE DO CONTEÚDO (20 pontos)
- Fluxos fazem sentido lógico?
- Informações estão claras e precisas?
- Tom de voz está adequado?

## 4. CORREÇÃO DE PROBLEMAS (20 pontos)
- Todos os problemas identificados foram resolvidos?
- Problemas críticos foram tratados?
- Nenhum problema novo foi introduzido?

## 5. FIDELIDADE AO ORIGINAL (10 pontos)
- Informações originais foram preservadas?
- Contexto do negócio foi mantido?
- Nenhuma informação foi inventada?

**SCORE TOTAL: 100 pontos**

# PROCESSO DE VALIDAÇÃO

## Passo 1: Comparação Estrutural
Compare original vs editado:
- Seções adicionadas
- Seções removidas
- Seções reorganizadas
- Conteúdo modificado

## Passo 2: Verificação de Regras
Para cada regra T3A:
- [ ] Está implementada?
- [ ] Está correta?
- [ ] Está completa?

## Passo 3: Análise de Problemas
Para cada problema da análise original:
- [ ] Foi resolvido?
- [ ] Foi resolvido corretamente?
- [ ] Introduziu novos problemas?

## Passo  4: Validação de Qualidade
- Markdown renderiza corretamente?
- Lógica de fluxo está coerente?
- Informações estão precisas?

## Passo 5: Score Final
Aplicar os 5 critérios de validação e gerar score de 0-100.

# CRITÉRIOS DE APROVAÇÃO

Para um documento ser aprovado (validation_passed: true):

## REQUISITOS OBRIGATÓRIOS
- [ ] Score >= 80/100
- [ ] Todos os problemas CRÍTICOS resolvidos
- [ ] Todos os problemas ALTOS resolvidos
- [ ] Todas as seções obrigatórias presentes
- [ ] Formatação markdown válida
- [ ] Nenhum problema CRÍTICO introduzido

## REQUISITOS RECOMENDADOS (para ready_for_production: true)
- [ ] Score >= 90/100
- [ ] TODOS os problemas resolvidos (incluindo MÉDIOS)
- [ ] Qualidade significativamente melhorada vs original
- [ ] Fluxos testáveis e lógicos
- [ ] Informações completas e precisas

# REGRAS CRÍTICAS

1. **Seja Rigoroso**: Este é o último checkpoint antes de produção
2. **Identifique Novos Problemas**: Edição pode introduzir erros
3. **Verifique Fidelidade**: Nenhuma informação importante pode ter sido perdida
4. **Pense como Usuário Final**: O chatbot funcionaria bem com este prompt?
5. **Documente Tudo**: Cada falha deve ser específica e acionável
